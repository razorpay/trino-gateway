## Query Domain Rules

- Query records are audit trail entries — created asynchronously by the router after proxying, not by API callers.
- Query text has VARCHAR(500) limit (increased from 255 in migration 20220107). Longer queries get truncated by MySQL.
- Query ID comes from the Trino server response, not generated by the gateway.
- FindBackendForQuery is the primary lookup method — exists as a dedicated RPC for the most common use case.
- Frontend pagination is client-side: fetches 1000 records, pages locally at 100/page.
- Validation in `validation.go` is entirely commented out.
